Tools & Technologies Used

  • HTML5 for semantic structure
  • CSS3 (Flexbox, Grid, responsive design)
  • Figma for inspecting spacing, typography, and layout details
  • Git & GitHub Pages for version control and deployment
  • VS Code as the main development environment
  • Google Fonts for typography
  • ARIA attributes to improve accessibility and screen‑reader support

Use of AI Assistance

I used AI (Microsoft Copilot) as a technical assistant, mainly to:

  • Clarify HTML, CSS, and accessibility concepts
  • Understand specific behaviors (flexbox, spacing, typography, ARIA)
  • Review my code and identify inconsistencies
  • Get explanations and best‑practice guidance

Important: I did not request or use AI‑generated component code. All implementation was written manually — AI served only as a guide, not as a code generator.

Reflections

What are you most proud of, and what would you do differently next time?

I’m proud of how much attention I put into the small details — matching spacing, typography, and layout exactly like the Figma design. I also focused a lot on accessibility, adding proper ARIA labels and improving the semantic structure of each component.

Next time, I’d plan my structure earlier instead of adjusting things as I go. I realized that having a clear layout in mind from the start saves time and avoids rewriting CSS later. I’d also organize my files better from the beginning so the project scales more smoothly.

What challenges did you encounter, and how did you overcome them?

My biggest challenges were understanding how fonts actually render in the browser, getting spacing to match the design exactly, and aligning elements without relying on flexbox. I also had to learn how ARIA works in real components, not just in theory.

I overcame these by breaking each problem down, testing small changes, and asking very specific questions. I used AI as a technical assistant — not to write code, but to clarify concepts, explain why something behaved a certain way, and help me spot inconsistencies. That made it easier to fix issues on my own and understand the “why” behind each solution.
